SACTCheck v0.17 validation report
=================================
Date: 18 July 2026

Scope
-----
- Conditional input visibility and requirement logic.
- Non-blocking optional/contextual inputs with explicit unassessed coverage.
- NCCP 00226 Version 9 weekly paclitaxel JSON shadow assessment.
- Preservation of existing generic assessments and FOLFOX shadow validation.
- One-file protocol publication workflow.

Conditional-input controls tested
---------------------------------
- Optional olaparib domains no longer block the core assessment.
- Irrelevant conditional fields remain hidden.
- Febrile-neutropenia follow-on fields appear and become mandatory only after the
  trigger is selected.
- Haemodialysis becomes mandatory only in severe renal impairment.
- Weekly-paclitaxel ANC-duration and toxicity follow-on fields activate only when
  their clinical trigger applies.
- Conditional definitions referring to undefined or self-referential inputs are
  rejected by the protocol validator.
- Missing optional values are not treated as normal; the result records the affected
  rule domains as unassessed and avoids an unconditional proceed result.

Weekly-paclitaxel boundaries tested
-----------------------------------
- ANC 1.00 versus 0.99 ×10^9/L.
- Platelets 90 versus 89 and 69 ×10^9/L.
- ANC below 0.5 for six versus seven days.
- Febrile neutropenia.
- Bilirubin 1.25, 1.26, 2.01 and 5.01 × ULN.
- Transaminases at 10 × ULN.
- Grade 2 and grade 3 neuropathy.
- Grade 2 non-neuropathy toxicity before and after recovery.
- Baseline neutrophil exclusion threshold.
- Percentage and fixed mg/m² dose calculations.

Clinical status
---------------
This remains a demonstration and validation prototype. The weekly-paclitaxel JSON is
based on NCCP 00226 Version 9 but remains pending independent consultant and
oncology-pharmacy line-by-line review. Software tests do not confirm source
interpretation, clinical accuracy, governance approval or authorisation for use.

Packaging-environment limitation
--------------------------------
Automated browser navigation to local HTTP/file URLs is blocked by the execution
environment. The JavaScript modules, inline scripts, JSON files, engines and rule
boundaries were tested programmatically, but the Live Server click-through remains
a required manual check before commit and push.
