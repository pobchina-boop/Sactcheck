SACTCheck v0.17 drop-in update
==============================

Purpose
-------
v0.17 fixes over-strict generated forms and adds weekly paclitaxel as the first
new regimen encoded against the final conditional schema. Core and applicable
conditional fields can block completion; optional or irrelevant fields do not.
Blank optional values are never interpreted as normal and remain visible in the
result as unassessed rule-coverage domains.

Before copying
--------------
1. Confirm the current v0.16.2 changes are committed and pushed on develop.
2. Keep the existing Sactcheck repository folder and its .git folder.

Install
-------
1. Extract SACTCheck_v0.17_drop_in_update.zip anywhere convenient.
2. Open the extracted SACTCheck_v0.17_drop_in folder.
3. Copy everything inside it into the root of the existing Sactcheck repository.
4. Choose "Replace the files in the destination".
5. Open index.html with VS Code Live Server.
6. Press Ctrl+F5 once in the browser.

Manual browser checks
---------------------
1. Confirm the catalogue badge reads "Version 0.17 · Conditional JSON assessments".
2. Open olaparib. Initially confirm febrile-neutropenia grade, haematological
   occurrence number and haemodialysis are not shown when their triggers do not apply.
3. Select febrile neutropenia. Confirm grade and occurrence fields appear and become
   mandatory.
4. Enter creatinine clearance below 30 mL/min. Confirm haemodialysis appears and
   becomes mandatory.
5. Leave optional Child-Pugh and CYP3A fields blank. Confirm the assessment runs,
   does not infer normality, and lists the omitted domains as unassessed.
6. Find Weekly paclitaxel. Confirm both the original "Open protocol assessment"
   button and the new "Open JSON shadow assessment" button remain available.
7. In the JSON shadow form, load demonstration values and run the assessment.
8. Test ANC 0.99, platelets 69, bilirubin 1.26 × ULN, bilirubin 5.01 × ULN,
   grade 2 neuropathy and grade 3 neuropathy.
9. Confirm Preview protocol JSON still validates and opens the authoring template.
10. Confirm FOLFOX legacy assessment and its shadow-comparison screen still work.

Automated tests
---------------
Open a terminal in the repository and run:

npm test

Expected results:
- 9 protocol-publishing tests passed.
- 13 generic assessment-engine tests passed.
- 7 conditional-input tests passed.
- 24 weekly-paclitaxel JSON tests passed.
- 17 FOLFOX shadow-comparison tests passed.
- 70 tests in total.

One-file publication workflow
-----------------------------
1. Copy protocols/_template/protocol-template.json.
2. Encode and clinically review the regimen.
3. Preview the JSON in SACTCheck and correct every validation error.
4. Place the reviewed JSON under the correct protocols/<tumour-site>/ folder.
5. Commit and push that single JSON file.
6. GitHub Actions validates the library, rebuilds protocols/index.json, runs all
   regression tests and commits the regenerated catalogue after a successful push.

Suggested commit message
------------------------
Add conditional inputs and weekly paclitaxel JSON v0.17

Clinical boundary
-----------------
Weekly paclitaxel remains in JSON shadow-validation mode. The original pathway stays
active pending consultant, oncology-pharmacy and governance review. Automated
structural validation and tests do not establish clinical authorisation.
