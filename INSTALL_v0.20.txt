DROP-IN UPDATE
Replace these files in the existing v0.19 project:
- js/protocol-loader.js
- tests/main-app-integration.test.js
- package.json
- index.html

What changed
- All published JSON protocol cards use one generic launch function.
- Generated catalogue cards and migrated legacy cards share the same launch binding.
- Launch buttons identify protocols by data-protocol-id; no protocol-specific launch code is required.
- Shadow-validation protocols remain hidden from routine catalogue launch.
- Adding a validated protocol to protocols/index.json is sufficient to create a launchable assessment card unless a legacy_card_id is supplied to reuse an existing card.
