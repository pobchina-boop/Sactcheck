SACTCheck v0.37.0 update checks
===============================

Before copying
--------------
1. Commit and push the current branch or create a backup.
2. Retain the existing repository .git folder.

Install
-------
1. Extract SACTCheck_v0.37.0_Platform_Standardisation.zip.
2. Copy the contents of the extracted Sactcheck folder into the repository root.
3. Replace existing files when prompted.
4. Run `npm run protocols:build`.
5. Run `npm test`.
6. Start VS Code Live Server and hard-refresh the browser.

Manual browser checks
---------------------
1. Confirm the badge reads "Version 0.37.0 · Platform standardisation".
2. Confirm Treatment Category appears beside the tumour-site and status filters.
3. Filter Breast + Endocrine and confirm tamoxifen, anastrozole, letrozole, exemestane and fulvestrant appear outside the chemotherapy section.
4. Search for an endocrine medicine while another treatment category is selected and confirm filter behaviour is explicit and predictable.
5. Open a high-risk regimen such as sacituzumab govitecan and confirm the supportive-care summary, mapping basis, validation warning and NCCP source link appear.
6. Open a phase-dependent pembrolizumab/taxane-carboplatin/AC regimen and confirm the antiemetic profile changes with the selected treatment phase rather than showing one static script.
7. Open a minimal-risk biologic and confirm it does not link to the old parenteral low-risk prescription sheet.
8. Open a protocol with diarrhoea/colitis, mucositis, neuropathy, rash or infusion-reaction grading and confirm Grade 0-4 definitions and "How to assess" guidance are displayed beside the selector.
9. Open a protocol with "other non-haematological toxicity" and confirm the app warns that the named CTCAE term must be identified before grading.
10. Open a protocol with categorical renal guidance and confirm ranges are selected rather than entered as an exact number.
11. Open a carboplatin regimen and confirm the continuous GFR/CrCl field remains numerical for Calvert dosing.
12. Confirm each protocol still provides one-click access to the official NCCP PDF.

Automated checks
----------------
Run:

npm run protocols:build
npm test

Expected v0.37.0 platform audit totals:
- 121 active regimen protocols
- 415 CTCAE-enabled fields
- 32 protocol-specific renal-band fields
- 27 exact carboplatin renal fields
- no duplicate protocol IDs, NCCP codes or normalised titles
- no active regimen placeholders

Clinical boundary
-----------------
Do not treat a passing software test as clinical validation. The complete library and the included 2021 local supportive-medicine sheets require consultant, oncology-pharmacy and governance review before deployment.
